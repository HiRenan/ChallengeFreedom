# Abertura de Tickets

Use este guia quando o atendimento precisar virar uma solicitação formal.

## Quando abrir um ticket

Abra um ticket quando houver:

- erro de acesso
- falha em sistema interno
- problema com VPN
- conta bloqueada
- permissão insuficiente para uma tarefa de trabalho

## Dados mínimos obrigatórios

Um ticket só pode ser aberto com estas informações:

- nome do solicitante
- e-mail de contato
- assunto
- descrição do problema

## Como escrever um bom assunto

O assunto deve resumir o problema em uma linha.

Exemplos bons:

- Acesso negado ao portal de folha
- VPN não conecta desde a manhã
- Conta de e-mail bloqueada

## Como escrever uma boa descrição

A descrição deve explicar:

- o que aconteceu
- quando começou
- em qual sistema ou recurso ocorreu
- o impacto no trabalho

## Exemplo de descrição útil

Não consigo acessar o portal de folha desde hoje pela manhã. Ao entrar com meu usuário, recebo mensagem de acesso negado. Preciso consultar o holerite ainda hoje.

## O que evitar

Evite abrir ticket com assunto genérico ou descrição vaga, como:

- Ajuda
- Não funciona
- Urgente

## Antes de enviar

Confira se o problema já não foi respondido na documentação de suporte. Se for apenas uma dúvida informacional, não é necessário criar ticket.
